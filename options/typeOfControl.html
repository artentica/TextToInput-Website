Type of control options

Variable

controleUniqueButton

Default

true

Possible Values

true or false

If you prefer to have a single field with buttons leave the default value, otherwise you can change it to false and to have buttons on each lines to edit the table / list, line by line. But if you do this remember to put the class ".actionCell" on each line or give it a new value. For more information refer to page Specific Class. You can see the "saveOnChange" option below.

<div id="MyTable">
  <div id="mainButtonDiv"></div>
  <table class="table">
    <thead>
      <tr>
        <th>Action</th>
        <th>First Name</th>
        <th>Last Name</th>
        <th>Username</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td class="actionCell"></td>
        <td>Vincent</td>
        <td>Riouallon</td>
        <td>@VRiouallon</td>
      </tr>
      <tr>
        <td class="actionCell"></td>
        <tdRémi</td>
        <td>Collignon</td>
        <td>Miton</td>
      </tr>
      <tr>
        <td class="actionCell"></td>
        <td>Gilles</td>
        <td>Biannic</td>
        <td>GillouBZH</td>
      </tr>
    </tbody>
  </table>
</div>
$('#MyTable').TextToInput({
    controleUniqueButton  : false
});

Variable

saveOnChange

Default

true

Possible Values

true or false

This option allows simply to save the changes when you press the Edit button of another line. This option is only active if "controleUniqueButton" is "false". If you decide to set it to "false" the "saveOnChange" option, the buttons of the other lines will simply be disabled until the validation or cancellation of the current changes.
