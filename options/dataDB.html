Data DB options

In this part we will see what is linked to the databases. Especially if the fields are specified in the table / list and if an identifier is given. TextToInput is done like so that it sends only the minimum necessary given. In the following example I will show you the result of the json, below the table so you can see the result.

To enable consideration of fields of the database in the information sent to you must enable the "PrecisedFieldDB".

Variable

PrecisedFieldDB

Default

false

Possible Values

true or false

You can change the name of the "data" attribute if you like, is in this attribute that we will put the name of fields of the database.

Variable

dataDB

Default

"name_DB"

Possible Values

String

If you specify the fields in the database, but not identifying the plugin will send all fields in each row with the key, the fields in the database. Each line that result in two objects, old fields to the match in the database, and the new changed fields to enter the values.

<div id="basicTableDataNoID">
  <div></div>
  <table class="table">
      <thead>
        <tr>
          <th>First Name</th>
          <th>Last Name</th>
          <th>Username</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td data-my_db_field="first_element">Vincent</td>
          <td data-my_db_field="it_is_my_name">Riouallon</td>
          <td data-my_db_field="username">@VRiouallon</td>
        </tr>
        <tr>
          <td data-my_db_field="first_element">Rémi</td>
          <td data-my_db_field="it_is_my_name">Collignon</td>
          <td data-my_db_field="username">Miton</td>
        </tr>
        <tr>
          <td data-my_db_field="first_element">Gilles</td>
          <td data-my_db_field="it_is_my_name">Biannic</td>
          <td data-my_db_field="username">GillouBZH</td>
        </tr>
      </tbody>
    </table>
</div>
$('#basicTableDataNoID').TextToInput({
    mainButtonDiv : "#basicTableDataNoID div:nth-child(1)",
    PrecisedFieldDB : true,
    SendJSON: function (json) { //I overload the send function of the data so that the outcome is displayed in a div
      $("#basicTableDataNoID .result").empty();
      $("#basicTableDataNoID .result").append(json);
    }
});

Warning: The red text does not resets because resetChanges() function is not called, go to the "SendJSON" to see how to remedy that. Or to the "resetChange" to see what does this function.

Here you have an example of a table with database fields and an identifier.

Variable

IDDB

Default

""

Possible Values

String

Warning: If you specify a class as being the identifier of your data you should specify it on each line. Give it a class is to declare the systematic use of an identifier in the current TextToInput object.

Trick: If you want to put the ID but it does not appear you can set "display:none".

Reminder: The class will be a IDDB "LockValue" unless you change the code the user can not change its value. For more information see the "Specific Class".

<div id="basicTableDataID">
  <div></div>
  <table class="table">
      <thead>
        <tr>
          <th>First Name</th>
          <th>Last Name</th>
          <th>Username</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td data-name_DB="first_element">Vincent</td>
          <td data-name_DB="it_is_my_name">Riouallon</td>
          <td class="IDDB">@VRiouallon</td>
        </tr>
        <tr>
          <td data-name_DB="first_element">Rémi</td>
          <td data-name_DB="it_is_my_name">Collignon</td>
          <td class="IDDB">Miton</td>
        </tr>
        <tr>
          <td data-name_DB="first_element">Gilles</td>
          <td data-name_DB="it_is_my_name">Biannic</td>
          <td class="IDDB">GillouBZH</td>
        </tr>
      </tbody>
    </table>
</div>
$('#basicTableDataNoID').TextToInput({
    mainButtonDiv : "#basicTableDataNoID div:nth-child(1)",
    PrecisedFieldDB : true,
    IDDB : ".IDDB",
    SendJSON: function (json) { //I overload the send function of the data so that the outcome is displayed in a div
      $("#basicTableDataNoID .result").empty();
      $("#basicTableDataNoID .result").append(json);
    }
});
